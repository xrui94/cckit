# Filesystem 测试说明

## 测试概述

本目录包含文件系统子库的测试用例，涵盖基础功能测试和性能测试。

## 测试文件

### 1. test_fs_basic.cpp
基础功能测试，验证文件系统的核心功能：

- **路径分隔符测试**：验证系统路径分隔符的正确性
- **路径存在检查测试**：验证 exists() 函数的功能
- **文件检测测试**：验证 isRegularFile() 和 isDirectory() 的功能
- **路径操作测试**：验证文件名、基名、扩展名等路径操作
- **当前目录测试**：验证 getCurrentDirectory() 的功能
- **绝对路径测试**：验证 getAbsolutePath() 的功能
- **通用路径测试**：验证 getGenericPath() 的路径标准化功能
- **文件操作测试**：验证文件复制、删除和大小获取功能
- **目录操作测试**：验证目录创建、删除和列表功能
- **确保路径存在测试**：验证 ensurePathExists() 的递归创建功能
- **文件列表过滤测试**：验证带扩展名过滤的文件列表功能
- **边界情况测试**：验证空路径、不存在的路径等边界情况的处理

### 2. test_fs_performance.cpp
性能测试，评估文件系统操作的性能：

- **文件创建性能测试**：评估批量创建文件的速度
- **文件读取性能测试**：评估文件存在检查和大小获取的性能
- **路径操作性能测试**：评估各种路径操作的性能
- **目录列表性能测试**：评估列出目录内容的性能
- **文件复制性能测试**：评估文件复制的速度和吞吐量
- **目录创建性能测试**：评估目录创建的性能

## 编译和运行

### 编译测试

```bash
# 配置项目
cmake -B build -DCCKIT_BUILD_TESTS=ON

# 编译
cmake --build build
```

### 运行所有测试

```bash
# 进入构建目录
cd build

# 运行所有测试
ctest --output-on-failure
```

### 运行单个测试

```bash
# 运行基础功能测试
./test_fs_basic

# 运行性能测试
./test_fs_performance

# 或者使用 CTest
ctest -R fs_basic
ctest -R fs_performance
```

## 注意事项

1. **性能测试结果可能因硬件和文件系统而异**
2. **测试运行时间可能较长，特别是性能测试**
3. **建议在 Release 模式下运行性能测试以获得准确结果**
4. **基础功能测试在 Debug 和 Release 模式下都应该通过**
5. **测试会创建临时文件和目录，运行后会自动清理**
6. **如果测试异常终止，可能需要手动清理临时文件**

## 扩展测试

如需添加新的测试用例，请遵循以下步骤：

1. 在 test 目录下创建新的测试文件
2. 在 CMakeLists.txt 中添加新的测试可执行文件
3. 更新本 README.md 文档
4. 确保新测试遵循现有测试的风格和命名约定
5. **重要**：确保测试在结束时清理所有临时文件和目录
6. **推荐**：使用 RAII 或析构函数确保资源清理
